Background

About the Project

The Problem

Organizations invest heavily in external security but frequently overlook risks from insiders who already have legitimate access to sensitive data. Insider threats exploit normal workflows — USB transfers, email attachments, and cloud uploads — to move data outside the organization.

There is no direct system event in Windows that signals when a user uploads a file to the cloud or sends it via email. USB transfers generate massive amounts of background noise that drowns out genuine transfer events.

As a result, organizations cannot reliably detect or reconstruct data exfiltration events using standard system logs, leaving a dangerous blind spot in their security posture.

Our Solution

Our project builds a modular, endpoint-based detection system that monitors and logs data exfiltration events across three distinct channels: USB drives, cloud storage uploads, and email attachments.

Rather than relying on direct system events, the system uses behavioral analysis — correlating file access patterns, process activity, and network events — to identify exfiltration with configurable confidence levels.

All three agents produce structured, investigator-friendly CSV logs with consistent fields and SHA-256 file hashes. This common format enables cross-channel correlation, allowing investigators to link the same file appearing in email, USB, and cloud logs — concrete evidence of multi-channel exfiltration.

System Design

System Architecture

Three independent detection agents feed a unified forensic layer, enabling cross-channel correlation via SHA-256 file hashes.

USB Transfer Monitor

  • Monitors CreateFile operations on non-system drives via ProcMon
  • Intelligent filtering of background noise to isolate real transfers
  • Captures timestamps, metadata, file copies, and SHA-256 hashes
  • Heartbeat system prevents unauthorized agent termination
ProcMon PML→CSV

Cloud Upload Detector

  • No direct OS event — uses behavioral correlation instead
  • Detects file read (CreateFile + ReadFile, ≥5 KB) followed by network sends within ~2 sec
  • Requires ≥3 network send events from the same browser process
  • Identifies Google, AWS, Microsoft OneDrive, and Dropbox services
Medium / High Confidence

Email Attachment Monitor

  • Dual-backend: Outlook COM (desktop) + Microsoft Graph API (web/365)
  • Polls every 5 seconds for outgoing emails with attachments
  • SHA-256 hash per attachment; recipient domain classification
  • Auto-forwarding rule detection via Graph API
~1,180 lines Python Thread-Safe

SHA-256 Forensic Bridge

Every file hash is computed across all three agents. If the same SHA-256 hash appears in email logs and USB logs, that is concrete, cryptographic evidence of multi-channel exfiltration — linking the same file across independent detection channels.

Unified CSV Logs

filename • path • timestamp • user • process • network • confidence • SHA-256

Technical Details

Implementation Details

Built with Python

  • Automates ProcMon for USB & cloud agents
  • PML → CSV pipeline and event extraction
  • ~1,180 lines for the email agent alone
  • Nuitka compilation: Python → native C binary

Structured CSV Logging

  • Filename & full file path
  • Timestamp, user, and process name
  • Network activity and confidence level
  • SHA-256 hash for forensic correlation

Threading & Concurrency

  • Concurrent Outlook COM + Graph API backends
  • Thread-safe data structures throughout
  • Automatic log rotation at 10 MB
  • Heartbeat system prevents forced termination

Zero-Config Deployment

  • Environment variable configuration
  • No source code changes needed to deploy
  • Modular agents run independently
  • Competitor domain list is configurable
Cloud Confidence Scoring

Medium: File read (≥5 KB) + ≥3 network send events from same browser process within ~2 seconds.

High: File read (≥10 KB) + ≥5 network sends. Services identified: Google, AWS, OneDrive, Dropbox.

Recipient Classification

Outgoing attachments classified by destination: Internal Personal Competitor External
Reports highest-risk classification when multiple recipients are present.

Auto-Forwarding Detection

Inspired by Dr. Robinson's insider threat research: insiders frequently enable mail auto-forwarding rules to silently exfiltrate all incoming email. Detected via Microsoft Graph API inbox rule inspection.

Evaluation

SWOT Analysis

Strengths

  • Multi-channel detection (email, USB, cloud) using behavioral analysis
  • Modular independent agents logging to a common CSV format
  • SHA-256 forensic bridge enables cross-channel correlation
  • Dual email backends cover both desktop and web Outlook
  • Auto-forwarding rule detection catches passive exfiltration

Weaknesses

  • Behavioral heuristics may miss edge cases or unusual workflows
  • USB agent depends on ProcMon, limiting stealth and scalability
  • Gmail not yet integrated as an email backend
  • No real-time alerting — requires manual log review
  • Confidence thresholds require tuning per environment

Opportunities

  • Kernel-level ETW file monitoring for desktop applications
  • Real-time alerting dashboard for immediate threat notification
  • SIEM integration (Splunk, Microsoft Sentinel)
  • Multi-account enterprise monitoring across organizations
  • Gmail API integration — architecture already supports it

Threats

  • Browser sandboxing prevents kernel-level webmail detection (Chrome’s memory-mapped I/O bypasses ETW)
  • OS and API updates could break existing integrations
  • Sophisticated insiders may use encrypted channels or personal devices
  • ProcMon detection by security-aware insiders
Roadmap

Future Work

Gmail API Integration

The system architecture already supports pluggable email backends. Adding a Gmail API backend would extend coverage to personal and mixed-use environments without restructuring existing code.

Kernel-Level ETW File Monitoring

Replacing ProcMon with direct Event Tracing for Windows (ETW) would provide stealthier, lower-overhead monitoring of file I/O events for desktop email clients and other processes, without requiring an external tool.

Real-Time Alerting Dashboard

A web-based dashboard that aggregates all agent logs in real time and triggers alerts when high-confidence exfiltration events or cross-channel hash matches are detected.

SIEM Platform Integration

Connector modules for Splunk and Microsoft Sentinel would allow the system’s structured CSV logs to feed directly into enterprise security information and event management platforms for centralized analysis.

Enterprise Multi-Account Monitoring

Scaling the system to monitor multiple user accounts concurrently across an organization, with centralized log aggregation and per-user behavioral baselines for more accurate anomaly detection.
